How BigCommerce Works · Reference

Glossary

The shared vocabulary for this course, in plain language, no ecommerce background assumed. Every lesson uses these terms exactly as defined here, and each definition follows the course's rule — the generic concept first, then how BigCommerce means it. Print it; keep it next to you. Grows as the course does.

The pipeline objects

TermDefinition
CartThe container stage: "a collection of items, prices, discounts, etc." that "does not contain customer-related data." Mutable and disposable — the pre-purchase basket. (Lesson 1)
CheckoutThe staging stage: a cart plus a billing address plus one or more consignments. A cart is promoted to a checkout by adding a billing address, and keeps the same ID. (Lesson 1)
OrderThe record stage: the durable object minted when an eligible checkout is converted. What fulfillment, refunds, and reporting act on. Created as incomplete. (Lesson 1)
Consignment"A list of physical products that will travel together to the purchaser," specifying "how those items can and will ship." Contains at least one line item, one address, and (before conversion) a selected shipping option. A shipment-in-waiting. (Lesson 1; full treatment later)

Handoffs & gates

TermDefinition
Billing addressThe customer commitment that promotes a cart to a checkout. The cart→checkout boundary — not payment. (Lesson 1)
selected_shipping_optionThe chosen shipping method on a consignment. A checkout with physical products cannot convert to an order until every consignment has one. The checkout→order gate. (Lesson 1)
incompleteAn order's status at API creation, before payment. (Lesson 1)
Awaiting FulfillmentThe status an order moves to automatically once payment succeeds. (Lesson 1)

The catalog

TermDefinition
ProductThe primary catalog entity — the marketing-page thing. Carries name, price, categories, and a type. Every product has at least one variant. (Lesson 2)
Product typephysical (has weight, ships to customers) or digital (downloadable files or services, no weight). Physical products force a consignment + shipping option at checkout. (Lesson 2)
Variant"An item as it sits on the shelf in the warehouse" — the saleable unit. Must have a SKU; tracks inventory; can override the product's price/weight/image. (Lesson 2)
Base variantA product with no options is its own single variant. So "a purchase is a variant" always holds, with no special case. (Lesson 2)
Variant optionA facet whose values combine to build variants — e.g. size (S/M/L) and color (R/G/B). Options auto-generate variants; each combination is a SKU. (Lesson 2)
Modifier"Any choice a shopper can make that doesn't affect the variant that's selected" — e.g. engraving text, a gift-wrap checkbox. Changes fulfillment, not the SKU; never tracks inventory. Can add cost. (Lesson 2)
SKUStock-keeping unit — the code identifying a specific variant, the level at which inventory is counted. (Lesson 2)

Line items

TermDefinition
Line itemOne row of a cart or order: a variant (or a custom item) plus a quantity, with name, price, and chosen options captured at add-time. A snapshot, not a live link to the catalog. (Lesson 3)
Line-item typesA cart's line_items splits rows into physical_items, digital_items, gift_certificates, and custom_items. (Lesson 3)
Custom itemA line item with no catalog link — carries its own sku, name, quantity, list_price; no product_id. For phone orders, one-offs, fees. (Lesson 3)
option_selectionsThe array used instead of variant_id when adding a product that has modifiers — encodes both the variant facet choices and the modifier choices together. (Lesson 3)
SnapshotThe principle that a line item (and an order's copied addresses/products) captures data at creation, so later catalog changes never rewrite existing carts or past orders. (Lessons 1, 3)

Pricing

TermDefinition
Price order of operationsThe fixed weighted stack that resolves a price: default → sale → variant → price list → modifiers → customer-group / bulk → discounts / coupons → tax (last). Read top (lowest weight) to bottom. (Lesson 4)
calculated_priceRead-only resolved shopper price after sale price and adjustment rules — the output of the stack, not "price minus a discount." (Lesson 4)
Price listA collection of price records. A price record is a per-variant, per-currency override (variant_id + price + currency). Missing variants fall back to catalog pricing. (Lesson 4)
Price list assignmentTies a price list to a customer group and/or sales channel — how one product carries retail and wholesale prices at once. (Lesson 4)
Customer groupA segment of customers; can attach a price list and a group discount. Group discounts do not apply on top of price-list prices. (Lesson 4)
Bulk pricingQuantity-break pricing (type price/percent/fixed over a quantity band). Skipped for SKUs that have price-list prices. (Lesson 4)
Tax-inclusive / exclusiveWhether a price already contains tax. Surfaced as entered_inclusive plus computed tax_inclusive / tax_exclusive. Tax applies last. (Lesson 4)

Discounts & promotions

TermDefinition
Sale priceA catalog-level price cut (sale_price), baked into the item and feeding calculated_price before a cart exists. Not a cart-time discount event. (Lessons 4–5)
PromotionA cart-time discount rule. Fires automatically or via a coupon code, per redemption_type. Owns the discount logic. (Lessons 5–6)
redemption_typeThe field deciding a promotion is AUTOMATIC (no code) or COUPON (a code unlocks it). Immutable after creation. (Lessons 5–6)
Two coupon systems(1) Promotions coupon code — a bare trigger; discount lives in the promotion. (2) Legacy v2 coupon — self-contained, carries its own type + amount. (Lesson 5)
RuleOne entry of a promotion's ordered rules: an action guarded by an optional condition. Rules run top-down; the first that applies a discount wins, the rest are skipped. (Lesson 6)
ConditionThe "when" of a rule: a cart condition (minimum_spend/minimum_quantity/item matchers) composed with and/or/not. (Lesson 6)
ActionThe "what" of a rule: cart_value, cart_items (with strategy), gift_item, shipping, or fixed_price_set; amount as fixed_amount or percentage_amount. (Lesson 6)

Fulfillment & the order

TermDefinition
available_shipping_optionsServer-generated fulfillment options for a consignment, computed from the destination address and item weight. Returned only when explicitly included. (Lesson 7)
Pickup consignment (BOPIS)A consignment fulfilled by in-store pickup — a pickupOption (location) plus line items, instead of an address + shipping option. (Lesson 7)
Two kinds of consignment update(1) select a shipping option; (2) change address / line items. Never both in one call — changing the address regenerates the options. (Lesson 7)
consignment_structure=objectQuery param for order consignments returning the supported object structure (shipping/pickups/downloads/email). Legacy array form deprecates 1 Feb 2026. (Lessons 7–8)
Order status / status_idOne of 15 lifecycle states (string + fixed numeric id). Payment transitions auto (Awaiting Payment → Awaiting Fulfillment); fulfillment/refunds are manual; status doesn't auto-revert. (Lesson 8)
ShipmentThe actual dispatch record (tracking number, carrier, items). Distinct from a consignment (the plan); partial shipments drive Partially Shipped → Shipped. (Lesson 8)
Sub-resourceOrder data returned as a {url, resource} pointer you expand: products, shipping_addresses, consignments, coupons, fees. (Lesson 8)

The APIs

TermDefinition
REST Management APIServer-side, store-wide API (snake_case). Owns authoritative catalog, pricing, promotions, and order data. Orders are a v2 API; catalog/pricing/promotions are v3. (Lessons 1, 4, 8)
REST Storefront APICustomer-scoped, client-side API (camelCase) for a shopper's own session — the same cart/checkout/consignment objects, accessed as the current customer. (Lesson 1)
GraphQL Storefront APIGraph access to storefront cart and checkout data alongside catalog data; useful for headless storefronts. (Lesson 1)
camelCase vs snake_caseStorefront API returns camelCase (billingAddress); Management API returns snake_case (billing_address). Same objects, different casing. (Lessons 1, 7)
