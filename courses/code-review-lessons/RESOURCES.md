# Resources — Code Review Lessons

The primary source for the *mistakes* is a real code review. The primary sources for the *principles* are the canonical writeups below. Each lesson cites the single best one.

## The originating review

- **A real code review of a sales-quote PDF endpoint.** The ground truth for every lesson's opening mistake, including one catch from an automated check. Tests were added in a later commit. (Reviewer names, org/repo, PR number, commit SHA, and URLs are stripped from all course files for this public repo. The private source lives in `lesson-briefs.md`, which must not be committed.)
- **`lesson-briefs.md`** (repo root, private) — the distilled briefs this course is built from, generated by `pr-to-lessons`. Keep it out of version control; it still carries the raw identifiers.

## Principle sources (one per lesson)

| Lesson | Principle | Primary source |
|---|---|---|
| L1 | DRY / single source of truth; copy fits its origin, not its destination | Hunt & Thomas, *The Pragmatic Programmer* — DRY. <https://pragprog.com/tips/> (Tip 15) |
| L2 | Put logic at the layer that owns it; presentation stays with rendering | Martin, *Clean Code* Ch.10 (Classes / SRP & cohesion); Django "where does logic go" — <https://docs.djangoproject.com/en/stable/misc/design-philosophies/> |
| L3 | Names & docstrings reveal intent, track scope | Martin, *Clean Code* Ch.2 (Meaningful Names) |
| L4 | Minimal explicit change; framework accessors over hand-rolled; explicit > implicit | PEP 20 — *The Zen of Python*. <https://peps.python.org/pep-0020/>. Django `Options.get_field` / `Field.validators` — verified in Django 6.0 source |
| L5 | Temporal coupling; co-locate what's always called together | Fowler, *Refactoring* — "Temporal Coupling" smell. <https://martinfowler.com/> ; c2 wiki "Temporal Coupling" |
| L6 | YAGNI — write for the current design | Fowler, "Yagni". <https://martinfowler.com/bliki/Yagni.html> |
| L7 | Test the new behaviour's contract even when neighbours lack tests | Martin, *Clean Code* Ch.9 (Unit Tests) + the Boy Scout rule (Ch.1) |

## Verified against real code (not recalled)

- **Django 6.0.6** in the project's venv — `Model._meta.get_field(name)` returns the `Field`; `Field.validators` = `[*default_validators, *_validators]`; `FileExtensionValidator(allowed_extensions=[...])`. These back L4's "the framework already knows it" claim.

## Community (for wisdom, when a judgment call is genuinely contested)

- The code review itself was the tightest feedback loop — the reviewers are the community here.
- For broader design-judgment debate: the codebase's own review history (search for prior copy-paste / layering discussions), and r/ExperiencedDevs for the "how much abstraction is too much" style questions L6 raises.
