Code Review Lessons

Twelve judgment lessons distilled from a real code review of a sales-quote PDF endpoint, turned into checks you run on your own diff. Not "learn Clean Code." Learn the tell that catches the mistake at self-review time. Twelve lessons, an interleaved quiz, a glossary, and a printable checklist.

Arc 1 · Adapting existing code

The PR's core sin: reusing what was already there without asking whether it fit.

L1 Copy a pattern only after checking it fitsAnnotate every pasted line with why it belongs; cut what fit the origin. The shared cache-key trap. → L2 Put logic at the layer that owns itTemplate-context on the model, rendering at the view: one concern split across two layers. →

Arc 2 · Clarity

Names, docstrings, and the size of the change.

L3 Names & docstrings track scopeWhen a comment out-names the variable, and when a docstring outlives the function's scope. → L4 Prefer the minimal, explicit changeSmallest diff that works; the framework's own accessor over a hand-built map; explicit over clever. →

Arc 3 · Coupling & horizon

Two mirror-image faults: splitting what belongs together, and joining in flexibility nothing asked for.

L5 Co-locate what's always called togetherTemporal coupling, found by reading the callers, not the function. → L6 Write for the current designYAGNI. Cut any hook added for a caller that doesn't exist yet. →

Arc 4 · Proof

Show the behaviour you built actually works — and that the tests proving it are honest.

L7 Test the new behaviourCover the observable contract even when the adjacent legacy code has no tests. → L8 Mock a real callable with a specAutospec so signature drift fails the test, instead of a false-green that ships the break. →

Arc 5 · Fidelity to the real thing

Stop coding against a remembered stand-in. Check the actual domain object, its full field set, and the real render environment.

L9 Put a constant next to its modelPlace a rule with the concept it governs, not the line that first reads it. Cohesion. → L10 Render every field the object carriesEnumerate the model's real fields; the dropped phone number is the one memory omits. → L11 Print CSS follows the render hostPhysical units and server-confirmed fonts; "looks right locally" is unverified. →

Arc 6 · Spec discipline

The gate before all the others: did you build what the ticket actually asked?

L12 Requirements checklist firstEnumerate the ticket before the first draft; review can't catch code that was never written. →
